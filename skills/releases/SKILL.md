# Releases

Use this skill to turn merged work into a release-ready summary.

## Release checklist
- Confirm scope included in the release
- Summarize notable changes
- Call out migrations or deployment implications
- Identify validation completed
- List known issues and follow-ups

## Versioning guidance
- Patch for fixes and internal improvements
- Minor for additive capabilities or meaningful workflow expansion
- Major for breaking changes to contracts or deployment behavior

## Output shape
1. Release summary
2. Included changes
3. Validation
4. Risks / operational notes
5. Next steps
